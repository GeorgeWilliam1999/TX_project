The Phase-5+ host **must** be one of: Allen CI nodes, an LHCb-tier
A5000 / A100 box, or a documented equivalent. Any other host requires
a `docs/decisions/NNNN-alternate-gpu-host.md` ADR before it can produce
acceptance numbers.

## MLflow

* Backend: local file `mlruns/` (gitignored) for Phases 0–4; a shared server can be added later without changing the run schema.
* Mandatory tags (refused-to-start at `tracking/check_tags.py`):
  `git_sha`, `git_dirty`, `allen_commit`, `protocol_sha`,
  `data_train_sha`, `data_val_sha`, `data_test_sha`, `splitter_sha`,
  `host`, `gpu_model`, `cuda_version`, `torch_version`, `cudnn_version`,
  `seed_numpy`, `seed_torch`, `seed_cuda`, `seed_dataloader`, `pythonhashseed`,
  `n_rk_steps`, `corrector_enabled`, `loss_recipe`, `precision`,
  `phase`, `purpose` ∈ {calibration, sweep, final, sanity}.

A run that starts without any of the above is killed by the helper
before training begins. This is enforced, not advised.

## Host capture (Phase 0)

`pins/host.txt` is generated by `scripts/capture_host.py` and contains:

```
hostname
uname -a
nvidia-smi --query-gpu=name,driver_version,memory.total --format=csv
nvcc --version
python -c "import torch; print(torch.version.cuda, torch.backends.cudnn.version())"
conda info --json | jq .conda_version,.platform
```

Re-run any time the host changes; the file is committed.
